Introduction
Standardize the metrics, its computation methodology & visual representation of this data that can be used to measure accuracy and efficiency of bug management process within SIV
Terminologies
# Terminology Definition How to compute
1 Bug throughput Overall days from open.new to complete/ reject status of a sighting in client_platf.bug tenant days_open value for the HSD-ES record which is in complete / rejected state
2 Triage throughput Days taken to triage a new sighting days_decimal_triage value for the record. If this is empty, it would mean the sighting status was never set to open.triage and this data can not be computed
3 Triage accuracy Correctness of triaged component which is measured once the sightings moves to complete state Ratio of HSDs with status = complete.product_changed HSDs where component is same as triaged_component and overall count of HSDs with status = complete.product_changed
4 DCR Ratio of valid defects to overall closed sightings Refer here
5 Sightings Any observations made on the platform which translates to a unique HSD-ES record ID NA
6 Defects Sightings that are confirmed bugs / issues HSDs with closed reasons marked as Green -> Refer here
7 Ageing Overall days from open.new of a sighting in client_platf.bug tenant days_open value for the HSD-ES record which is not in complete / rejected state
8 backlog Count of open + implemented sightings Count of open + implemented sightings
9 Silicon defect A HSD-ES record that is resolved as an Intel Silicon issue status = complete & closed_reason = internal_si_bug
10 Non Intel issues Issues not related to Internal HW/SW/FW status = complete & closed_reason = customer* OR 3rd_paty OR os_issue
